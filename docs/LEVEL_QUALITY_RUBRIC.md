# Level Quality Rubric v1

## A. 硬指标（Gate）

1) **bestWinRate >= 0.80**
- 解释：最优路线应稳定赢。
- 默认阈值：0.80（可调 0.70~0.95）。
- 例子：0.83 通过，0.72 失败。

2) **primaryDelta >= 0.25**
- 解释：关键点错一步应显著掉胜率。
- 默认阈值：0.25（可调 0.15~0.40）。
- 例子：best=0.82, second=0.54，delta=0.28 通过。

3) **maxDecisionPoints <= 3**
- 解释：MVP 解谜不应过度分叉。
- 默认阈值：3（可调 2~5）。

4) **maxCIWidth <= 0.20**
- 解释：统计不确定性不能太大。
- 默认阈值：0.20（可调 0.10~0.30）。

5) **uniqueScore >= 0.60**
- 解释：近最优走法不能太多，保证“解谜感”。
- 默认阈值：0.60（可调 0.40~0.85）。

6) **pvLength <= 20 plies**
- 解释：避免超长算牌题。
- 默认阈值：20（可调 12~30）。

7) **explainabilityScore >= 0.65**
- 解释：关键点可被复盘文案解释。
- 默认阈值：0.65（可调 0.50~0.80）。

## B. 软指标（Fun 排序）

- 关键点尽早出现（primaryDecisionPly <= 8）
- 常见错误 move 自然出现（top3 内存在明显错误分支）
- 牌型与策略标签多样性
- 爽点标签覆盖：控先手 / 让牌 / 逼炸 / 卡2 / 卡A / 王处理
